# CONTEXT: AI Animation Maker

A professional bridge for video editors to create and fine-tune **Hyperframes** (HTML/GSAP-based animations). The tool automates complex animation logic while providing a familiar "Inspector" interface for precision control.

## Glossary

### Core Entities

- **Composition**: The root project container. It includes the `index.html`, `meta.json`, and any sub-compositions. It defines the canvas dimensions and overall timeline.
- **AI Animator**: The backend engine that translates natural language prompts or structured scripts into Hyperframes-compliant code (HTML/CSS/GSAP).
- **Inspector**: The UI window used by editors to manually adjust properties. It dynamically generates controls based on metadata found in the Composition.
- **Inspector Hook**: A metadata decorator (e.g., `data-inspect="x,y,opacity"`) added to HTML elements to signal that they are controllable by the Inspector.
- **Clip**: A timed element within a Composition, defined by `data-start` and `data-duration`.
- **Relinking Manager**: A system that tracks absolute paths of referenced assets. It detects "Media Offline" states when files are moved or renamed and provides a UI for the user to locate and reconnect them.
- **Simplified Playback Timeline**: A visual UI component representing clips as horizontal bars. It allows editors to intuitively adjust `data-start` and `data-duration` properties via dragging and resizing.

### Technical Primitives

- **Hyperframes**: The rendering format. Uses standard web technologies (HTML, CSS, JS) with specific data-attributes for timeline management.
- **GSAP Timeline**: The motion engine. Must be initialized as `paused: true` and registered to `window.__timelines` for deterministic rendering.
- **Renderer**: The system (utilizing Node.js and FFmpeg) that captures frames from the browser-based Composition and encodes them into a video file.
- **Configuration-Driven Sync**: The architectural pattern where animation parameters are stored in a structured configuration object (e.g., JSON or a JS constants block) rather than hard-coded in the GSAP logic. The Inspector modifies this object, and the GSAP timeline consumes it, ensuring maintainability and scalability.
- **Timeline Factory Pattern**: A structural pattern where GSAP timelines are generated by a dedicated function (a "Factory") that accepts a configuration object as input. This allows the app to re-generate the entire animation state instantly when configuration changes are made in the Inspector.
- **Dual-Mode Preview**: A viewing system featuring an **Interactive Mode** (high-performance real-time UI) and a **Render Preview** (frame-accurate representation of the final export).
- **Asset Management (Referenced)**: A professional workflow where media is referenced from its original location on the disk rather than copied into the project folder. This avoids storage redundancy but requires a robust "Relinking" mechanism for when files are moved.

## Tech Stack & Environment

- **Platform**: Tauri (Desktop). Chosen for its lightweight footprint and high performance.
- **Frontend**: React. Used for the high-performance Inspector UI and Project Management views.
- **Sidecars**: Bundled standalone Node.js and FFmpeg binaries. These are included in the app package to ensure a zero-config, "out-of-the-box" experience without requiring the user to install them separately.
- **AI Animator**: Integration with external LLM APIs (e.g., Claude) to generate Hyperframes code based on user prompts.
- **Bridge**: Tauri's Rust-based core handles the orchestration between the React UI, the local file system, and the Sidecar binaries (Node/FFmpeg).
